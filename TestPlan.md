<a name="introduction"/>

## Введение

Данный документ описывает план тестирования веб-приложения **Siam Lacorns**. Цель тестирования – проверка соответствия реального поведения программы и её ожидаемого поведения, описанного в документации к проекту.  
Тестирование направлено на проверку ключевого функционала: регистрацию пользователей, управление коллекциями лако́рнов, просмотр каталога лако́рнов, поиск и фильтрацию, а также систему избранного.

<a name="items"/>

## Объект тестирования

В качестве объектов тестирования выделяются следующие функциональные возможности:

* Регистрация нового пользователя и аутентификация;
* Создание, редактирование и удаление пользовательских коллекций;
* Добавление и удаление лако́рнов из коллекций;
* Просмотр каталога всех лако́рнов;
* Поиск и фильтрация лако́рнов по различным атрибутам;
* Добавление лако́рнов в избранное;
* Просмотр детальной информации о лако́рне.

<a name="quality"/>

## Атрибуты качества

1. **Функциональность**:
   * функциональная полнота: приложение должно реализовывать все заявленные функции;
   * функциональная корректность: все функции должны работать в соответствии с требованиями.
2. **Удобство использования**:
   * интерфейс должен быть интуитивно понятным и удобным;
   * основные функции должны быть доступны не более чем за 2–3 действия.
3. **Надежность**:
   * система должна сохранять корректное состояние данных после выполнения операций.
4. **Производительность**:
   * время загрузки каталога лако́рнов не должно превышать 3 секунд;
   * операции с коллекциями должны выполняться без заметных задержек.

<a name="risk"/>

## Риски

К рискам можно отнести:
* возможную потерю данных пользовательских коллекций при сбоях системы;
* проблемы с производительностью при большом количестве лако́рнов в каталоге;
* некорректное отображение интерфейса на различных браузерах и устройствах;
* возможность дублирования лако́рнов в коллекциях из-за ошибок логики;
* проблемы с поиском и фильтрацией из-за неоптимальных запросов к базе данных.

<a name="features"/>

## Аспекты тестирования

В ходе тестирования планируется проверить реализацию основных функций веб-приложения. Аспекты, подвергаемые тестированию:

* Регистрация и аутентификация пользователя
* Управление пользовательскими коллекциями
* Работа с каталогом лако́рнов
* Поиск и фильтрация
* Система избранного

### Регистрация и аутентификация пользователя
Необходимо протестировать:
* регистрацию нового пользователя с валидными данными;
* обработку ошибок при вводе невалидного email или слабого пароля;
* вход с корректными учетными данными;
* обработку ошибок при неверном пароле или email;
* корректное завершение сессии при выходе.

### Управление пользовательскими коллекциями
Необходимо протестировать:
* создание новой коллекции с валидным названием и описанием;
* редактирование существующей коллекции;
* удаление коллекции с подтверждением действия;
* обработку попытки создания коллекции с дублирующим названием.

### Работа с каталогом лако́рнов
Необходимо протестировать:
* просмотр полного каталога лако́рнов;
* просмотр детальной информации о конкретном лако́рне;
* добавление лако́рна в пользовательскую коллекцию;
* удаление лако́рна из коллекции.

### Поиск и фильтрация
Необходимо протестировать:
* поиск лако́рнов по названию;
* фильтрацию по различным атрибутам (тип, редкость, цвет и т.д.);
* обработку ситуации, когда по запросу ничего не найдено;
* сброс фильтров к исходному состоянию.

### Система избранного
Необходимо протестировать:
* добавление лако́рна в избранное;
* удаление лако́рна из избранного;
* просмотр списка избранных лако́рнов;
* предотвращение дублирования в избранном.

<a name="nonfunctional"/>

## Нефункциональные требования

* Интерфейс должен быть удобным и понятным для пользователя.
* Интерфейс должен быть адаптивным и корректно отображаться на экранах различных устройств (ПК, планшеты, мобильные устройства).
* Все элементы интерфейса должны иметь корректные названия и подсказки.
* Время отклика при поиске лако́рнов не должно превышать 3 секунд.
* Время загрузки страниц не должно превышать 5 секунд.
* Система должна поддерживать одновременную работу не менее 100 пользователей.

<a name="approach"/>

## Подходы к тестированию

Для тестирования будет использован **ручной подход**.  
Проверка функций будет производиться через веб-интерфейс приложения. Каждый сценарий будет включать входные данные, ожидаемый результат и фактический результат. Сравнение ожидаемого и фактического поведения позволит определить корректность реализации функций.

<a name="conclusion"/>

## Выводы

Данный план тестирования охватывает ключевой функционал веб-приложения **Siam Lacorns**. Успешное прохождение всех тестов не гарантирует идеальную работу системы на всех возможных платформах и в любых условиях, однако подтверждает её корректное функционирование в рамках заявленных требований.  
В случае выявления ошибок в ходе тестирования они будут зафиксированы, после доработки проекта тестирование будет проведено повторно.
